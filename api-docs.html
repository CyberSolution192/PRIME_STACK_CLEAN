Prime Connect Developer API

Simple, powerful, and reliable data bundle integration

The Prime Connect Developer API lets you integrate data bundle purchases directly into your own website or application. Your customers pay you — you call our API — we deliver the data to their phone. Your wallet is automatically charged for each successful order.

Fast Integration Secure Wallet-Based

Get Started in 3 Steps

1

Get Your API Key

Go to Dashboard → API Access and generate your personal key

2

Top Up Your Wallet

Fund your wallet on the dashboard — each API order deducts automatically

3

Make Your First Request

Use the examples below to place your first data bundle order

Authentication

Every request must include your API key in the X-API-Key header. No session, no login flow.

# Include this header in every request
X-API-Key: pk_live_your_api_key_here

Server-side only. Never expose your API key in frontend JavaScript, mobile apps, or public repositories. All calls must be made from your backend server.

Your key starts with pk_live_ and is shown only once when generated. Store it securely immediately. If lost, revoke it and generate a new one from Dashboard → API Access.

Base URL

All requests are POST to this base URL:

https://primeconnect.site/api/

Append the endpoint name to the base URL for each request. Full examples:

# Check balance
POST https://primeconnect.site/api/api-balance

# Place an order
POST https://primeconnect.site/api/api-order

# Check order status
POST https://primeconnect.site/api/api-order-status

# Get available bundles
POST https://primeconnect.site/api/api-get-bundles

# Get transaction history
POST https://primeconnect.site/api/api-get-transactions

Rate Limits

Limits are per API key using a rolling 60-second window. Exceeding the limit returns 429 with a retry_after_seconds field.

EndpointLimitWindow
api-balance60 requestsPer minute
api-order30 requestsPer minute
api-order-status60 requestsPer minute
api-get-bundles60 requestsPer minute
api-get-transactions60 requestsPer minute
 
// 429 Rate Limit Exceeded
{
  "status": false, "statusCode": 429,
  "message": "Rate limit exceeded. Retry after 47 seconds.",
  "retry_after_seconds": 47
}

Error Handling

All responses follow a consistent structure. Always check the status field first.

// All responses — success and failure — share this shape
{
  "status":     true | false,
  "statusCode": 200,
  "message":    "Human readable description",
  "payload":    { /* present on success only */ }
}
CodeMeaningCommon Cause
200SuccessCheck status field in body
400Bad RequestMissing/invalid fields, insufficient balance, frozen wallet
401UnauthorisedMissing or invalid X-API-Key header
403ForbiddenAPI key revoked — generate a new one from the dashboard
404Not FoundBundle unavailable for this network/size, or order not found
409ConflictConcurrent wallet update — safe to retry after a short delay
429Too Many RequestsRate limit exceeded — wait retry_after_seconds
500Server ErrorUnhandled error — contact support if it persists

Common Error Responses

// 401 — Invalid API key
{ "status": false, "statusCode": 401, "message": "Invalid API key" }
// 400 — Insufficient balance
{ "status": false, "statusCode": 400, "message": "Insufficient balance. Required: GH₵22.00, Available: GH₵10.00" }
// 400 — Invalid phone
{ "status": false, "statusCode": 400, "message": "Invalid phone number. Must be exactly 10 digits, no country code." }
// 404 — Bundle not found
{ "status": false, "statusCode": 404, "message": "Bundle not available for this network and size" }

API Endpoints

POST /api-balance 60 req/min

Returns your current wallet balance and frozen status. No request body required.

const res = await fetch(
  'https://primeconnect.site/api/api-balance',
  {
    method: 'POST',
    headers: {
      'X-API-Key':     process.env.PRIME_API_KEY,
      'Content-Type': 'application/json',
    },
  }
);
const data = await res.json();
console.log(data.payload.balance); // e.g. 178.50
curl -X POST \
  https://primeconnect.site/api/api-balance \
  -H "X-API-Key: pk_live_your_key" \
  -H "Content-Type: application/json"
import requests

res = requests.post(
    "https://primeconnect.site/api/api-balance",
    headers={"X-API-Key": "pk_live_your_key"},
)
print(res.json()["payload"]["balance"])

Success Response 200

{
  "status": true, "statusCode": 200,
  "message": "Balance retrieved successfully",
  "payload": {
    "balance":  178.50,
    "currency": "GHS"
  }
}
POST /api-order 30 req/min

Places a data bundle order for your customer. Deducts from your wallet immediately.

Request Parameters

ParameterTypeRequiredDescription
phoneStringYes10-digit number, no country code. e.g. 0241234567
sizeIntegerYesBundle size in GB. Must match an active bundle. e.g. 1, 5, 10
networkStringYesmtn, telecel, or airteltigo (case-insensitive)
 
const res = await fetch(
  'https://primeconnect.site/api/api-order',
  {
    method: 'POST',
    headers: {
      'X-API-Key':     process.env.PRIME_API_KEY,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      phone:   '0241234567',
      size:    5,
      network: 'mtn',
    }),
  }
);
const data = await res.json();
if (data.status) {
  console.log('Order ref:', data.payload.order_reference);
  console.log('New balance:', data.payload.new_balance);
  console.log('Status:', data.payload.status);
}
curl -X POST \
  https://primeconnect.site/api/api-order \
  -H "X-API-Key: pk_live_your_key" \
  -H "Content-Type: application/json" \
  -d '{"phone":"0241234567","size":5,"network":"mtn"}'
res = requests.post(
    "https://primeconnect.site/api/api-order",
    headers={"X-API-Key": "pk_live_your_key"},
    json={"phone": "0241234567", "size": 5, "network": "mtn"},
)
data = res.json()
print(data["payload"]["order_reference"])

Success Response 200

{
  "status": true, "statusCode": 200,
  "message": "Order placed successfully",
  "payload": {
    "order_reference":  "API-M9K2D4-IJ78KL",
    "network":          "mtn",
    "size":             5,

POST /api-get-bundles 60 req/min

Returns all available data bundles and their prices. Use this to build your product listing dynamically rather than hardcoding prices.

Request Parameters

ParameterTypeRequiredDescription
networkStringOptionalFilter by network: mtn, telecel, or airteltigo. Omit to return all.
 
const res = await fetch(
  'https://primeconnect.site/api/api-get-bundles',
  {
    method: 'POST',
    headers: {
      'X-API-Key':     process.env.PRIME_API_KEY,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ network: 'mtn' }), // optional
  }
);
const { payload } = await res.json();
payload.bundles.forEach(b => console.log(`${b.size}GB — GH₵${b.price}`));
curl -X POST \
  https://primeconnect.site/api/api-get-bundles \
  -H "X-API-Key: pk_live_your_key" \
  -H "Content-Type: application/json" \
  -d '{"network":"mtn"}'
res = requests.post(
    "https://primeconnect.site/api/api-get-bundles",
    headers={"X-API-Key": "pk_live_your_key"},
    json={"network": "mtn"},  # optional
)
bundles = res.json()["payload"]["bundles"]

POST /api-get-transactions 60 req/min

Returns your API order history with pagination support. Only returns orders placed via the API — not dashboard purchases.

Request Parameters

ParameterTypeRequiredDescription
limitIntegerOptionalRecords per page. Default 10, max 50
offsetIntegerOptionalPagination offset. Default 0
 
const res = await fetch(
  'https://primeconnect.site/api/api-get-transactions',
  {
    method: 'POST',
    headers: {
      'X-API-Key':     process.env.PRIME_API_KEY,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ limit: 10, offset: 0 }),
  }
);
const { payload } = await res.json();
console.log(`${payload.count} of ${payload.total} orders`);
curl -X POST \
  https://primeconnect.site/api/api-get-transactions \
  -H "X-API-Key: pk_live_your_key" \
  -H "Content-Type: application/json" \

payload = res.json()["payload"]

Best Practices

Secure your API key

Store it as an environment variable. Never expose it in client-side code or version control. If compromised, revoke it immediately from Dashboard → API Access.

Server-side calls only

All API requests must originate from your backend. Never call the API directly from a browser or mobile client.

Monitor your balance

Call api-balance before bulk orders to confirm sufficient funds and a non-frozen wallet.

Validate input before sending

Validate phone numbers (10 digits, no country code) and bundle sizes on your side before calling the API to avoid unnecessary 400 errors.

Handle errors gracefully

Always check status in every response. For 409 conflicts, retry after a short delay. For 500 errors, contact support before retrying.

Use Get Bundles dynamically

Fetch bundles via api-get-bundles to build your product listing dynamically. Prices may change — never hardcode them in your application.

Need Help?

Verify your key starts with pk_live_, ensure your wallet has sufficient balance, and check error messages for specific details. Contact support through your dashboard if issues persist.
